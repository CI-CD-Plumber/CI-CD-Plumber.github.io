What We Offer - I

  • Smell categorization
    • based on detection source
    • based on type of smell
    • using methods on how to find
  • Proposing fixes
    • based on the methods of how to find (the source of the issue)
  • Data visualization to detect which code change increased build time

Basically, a tool that detects smells and propose fixes in CI/CD pipelines configured in Github Actions. Planned to work with Github Actions as an extension and a separate application.

What We Offer - II

Analysis and grading of the configuration of CI/CD pipeline based on a grading system.

Grading system categorizes smells into three:

  • mild, (least error weight)
  • moderate, (medium error weight)
  • severe. (highest error weight)

Each smell decreases the completeness of the pipeline hence decreasing the score. Users can see the score and analysis of their pipeline configurations.
